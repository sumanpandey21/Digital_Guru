DMA Working Mechanism

Step-by-Step Process: I/O Device Reads Data from Memory

Phase 1: Initialization (By CPU)

  • DMA Setup by CPU:
    • Source Address Register (SAR): Set to the memory location where the data resides.
    • Destination Address Register (DAR): Set to the I/O device address where the data will be sent.
    • Transfer Count Register (TCR): Set to the number of data units to transfer.
  • DMA Mode Selection:

    The CPU informs the DMA Controller of the transfer mode (read from memory, write to I/O).

Phase 2: DMA Request and Bus Control

  • I/O device generates a DMA Request signal to indicate a need to read data from memory.
  • DMA Controller asserts the Bus Request signal to the CPU to gain control of the buses.
  • CPU acknowledges by asserting the Bus Grant signal, giving control to the DMA Controller.

Phase 3: Address Selection and Data Fetch

  • SAR Address Selection: Address Select logic activates SAR and places the memory address on the Address Bus.
  • Read Operation: The memory responds by placing data onto the Data Bus.

Phase 4: Data Transfer to I/O Device

  • Address Select logic activates DAR, placing the I/O device address on the Address Bus.
  • Write Operation: Data on the Data Bus is transferred to the I/O device.

Phase 5: Register Updates

  • Registers Updated:
    • SAR increments to the next memory location.
    • DAR updates for the next I/O device location.
    • TCR decrements to reflect remaining data.
  • Check TCR:
    • If TCR > 0, process repeats.
    • If TCR = 0, transfer is complete.

Phase 6: Completion

  • DMA Acknowledge: Indicates transfer completion to the I/O device.
  • Bus Control Return: CPU regains control of the buses.

Summary of Signals and Components Involved

  • SAR, DAR, TCR: Manage addresses and transfer count.
  • Read/Write Signals: Control data flow between memory and I/O.
  • Address Select: Activates SAR or DAR appropriately.
  • DMA Request/Acknowledge: Initiates and completes DMA operations.

This process ensures efficient data transfer between memory and I/O devices without burdening the CPU.
